Логика работы locations:

1. Приходит запрос вида http://name.ru/folder_1/folder_2/folder_3/folder_4/

2. У нас есть несколько locations:
	a) locations / {}
	b) locations /folder_1/ {}
	c) locations /folder_1/folder_2/ {}

3. Видим что подходят все три locations

4. Ищем запрошенную папку в каждом. Нам нужны папки folder_3/folder_4/

5. После поиска в каждом locations понимаем что во всех 3х случаях есть такие папки

6. Выбираем самый длинный locations:     c) locations /folder_1/folder_2/ {};

7. Ищем файл индекса
	а) нашли
		отправляем его в ответ
	б) не нашли
		запрашиваем автоиндекс
			автоидекс включен
				выводим список содержимого дирректории
			автоиндекс выключен
				404
